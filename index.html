Ambulance Priority System

A signal preemption system that clears green lights ahead of an approaching ambulance and restores normal timing once it passes — reducing intersection delay on emergency routes. Two alternate street routes are mapped, each with its own turns and signal sequence; switch routes to see the preemption logic re-run against a different path.

How Preemption Works
01

Detect

Onboard GPS reports the ambulance's position, heading and speed to the priority server every tick.

02

Predict

The server calculates estimated time of arrival to each upcoming signal along the active route, following every turn in the street network.

03

Request

Once ETA drops under the trigger threshold, a priority request is sent to that intersection's controller.

04

Preempt

Conflicting phases end with a safe amber clearance, then the signal switches and holds green.

05

Restore

After the ambulance clears the intersection, control returns to the normal signal cycle.

This demo runs entirely client-side: vanilla JavaScript drives a tick loop that stands in for the priority server, and inline SVG renders the road network, signals and vehicle in real time — no backend required to see the logic work.

Each route is stored as an ordered list of waypoints. The simulation walks the ambulance along that polyline by distance travelled rather than a single x-coordinate, so the vehicle's position, heading and rotation update correctly through every turn. Intersections are anchored to a distance along the route rather than a fixed screen position, which is what lets the same trigger/preempt/restore logic work unmodified on either route.

A deployed version of this system would replace the simulated GPS feed with a real vehicle-to-infrastructure (V2I) channel — GPS plus an RF beacon such as an Opticom/GTT-style emitter, or an MQTT feed into the municipal signal network — and route choice would come from live dispatch/routing data rather than a dropdown.

Ambulance Priority System — interactive portfolio demo. Signal timings and street geometry are illustrative, not derived from a live traffic network.
